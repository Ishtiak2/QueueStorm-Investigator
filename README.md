# QueueStorm Investigator

FastAPI service that analyses bKash customer complaints and returns a structured
investigation response (Section 6 of the SUST Hackathon Preliminary problem
statement).

## Endpoints

| Method | Path             | Purpose                                                |
|-------:|------------------|--------------------------------------------------------|
| GET    | `/health`        | Liveness probe. Returns `{"status":"ok"}` in <1 s.     |
| POST   | `/analyze-ticket`| Structured investigation. 200 with full payload.       |
| GET    | `/docs`          | Swagger UI (auto-generated by FastAPI).                |
| GET    | `/redoc`         | ReDoc UI.                                              |
| GET    | `/openapi.json`  | OpenAPI 3 schema.                                      |

`POST /analyze-ticket` accepts JSON, returns the 12 Section 6 fields, and
completes well under the 30-second request budget.

## Local development

```bash
git clone https://github.com/Ishtiak2/QueueStorm-Investigator.git
cd QueueStorm-Investigator
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

Open http://127.0.0.1:8765/docs for the interactive Swagger UI.

## Deploying on Render (recommended)

The repo ships a [Render Blueprint](https://render.com/docs/blueprint-spec)
at `render.yaml` so the service can be deployed in two clicks:

1. Sign in to https://render.com with GitHub.
2. **New в†’ Blueprint** в†’ select `Ishtiak2/QueueStorm-Investigator`.
3. Render reads `render.yaml` and provisions the service automatically:
   - Runtime: Python 3.11
   - Plan: Free (sleeps after 15 min idle, wakes on first request)
   - Build: `pip install --upgrade pip && pip install -r requirements.txt`
   - Start: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - Health check: `GET /health`
4. Wait ~2 min for the first build. The URL will be
   `https://queuestorm-investigator.onrender.com`.

### Manual Render deploy (no Blueprint)

If you'd rather configure by hand:

1. **New в†’ Web Service в†’ Connect repo** `Ishtiak2/QueueStorm-Investigator`.
2. **Runtime**: Python 3.
3. **Build Command**: `pip install --upgrade pip && pip install -r requirements.txt`.
4. **Start Command**: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.
5. **Health Check Path**: `/health`.
6. **Instance Type**: Free.
7. Click **Create Web Service**.

Render assigns a `PORT` env var; the start command reads it via `$PORT`.

### Verifying the live deployment

```bash
# Health
curl https://queuestorm-investigator.onrender.com/health
# -> {"status":"ok"}

# Analyze a sample ticket
curl -X POST https://queuestorm-investigator.onrender.com/analyze-ticket \
  -H "Content-Type: application/json" \
  -d @Should_Be_Hidden/SUST_Preli_Sample_Cases.json   # adapt as needed
```

First request after the service has slept may take ~30 s to wake the
free-tier instance вЂ” subsequent requests are fast.
